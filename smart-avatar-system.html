OVERVIEW

概要

AIスマートアバター情報伝達システムは、CSV・JSON・DBなどのデータをもとに、 アバターが案内を行う仕組みです。 単に文章を表示するのではなく、音声と映像を使って内容を伝えることで、 より直感的に理解しやすい情報提示を目指しています。

ポートフォリオの中では、インターンでの経験から現在の実務へとつながっている代表的なテーマのひとつです。

BACKGROUND

背景

インターンでは、学内業者の営業情報や大学の魅力を分かりやすく伝える仕組みに取り組みました。 その中で、単に情報を並べるだけでなく、「どう見せると伝わるか」を考えることの重要性を感じました。

現在取り組んでいるアバター情報伝達システムは、その延長線上にあるものです。 数値や文章だけでは伝わりにくい案内情報を、 アバターによる映像と音声に変換して届けることで、 より理解しやすい案内の形を実現しようとしています。

PROBLEM

解決したい課題

文章や数値だけでは伝わりにくい

一覧表示や数値だけでは、短時間で内容を理解しづらい場面があります。

案内情報をもっと自然に伝えたい

利用者に対して、一方的な表示ではなく、案内として受け取りやすい形が求められていました。

更新・運用しやすい仕組みが必要

現場で使うには、CSV・JSON・DBなどのデータから柔軟に内容を変えられることが重要でした。

SOLUTION

システムで行っていること

データをもとに案内内容を生成

CSV・JSON・DBなどのデータをもとに、必要な案内内容を整理し、シナリオとして扱える形にします。

アバターによる映像・音声出力

テキスト情報をそのまま見せるのではなく、アバターによる映像と音声に変換して案内します。

用途に応じた表現切り替え

案内内容に応じて、複数のシナリオや見せ方に対応できる構成を意識しています。

更新しやすい運用

元データを更新することで内容を差し替えられるため、継続的な運用にも向いています。

FLOW

処理の流れ

  1. CSV / JSON / DB などから必要な情報を取得
  2. 案内内容に合わせてシナリオやテキストを生成
  3. アバター用の映像・音声コンテンツを作成
  4. 利用者向けに案内として出力

POINT

工夫した点

情報を「見せ方」まで含めて考える

ただデータを表示するのではなく、利用者にどう届けると理解しやすいかを意識して取り組みました。

データ連携を前提にした設計

単発のコンテンツではなく、CSV・JSON・DBなど複数のデータソースから扱える構成を意識しました。

案内用途に合わせた情報整理

何を、どの順序で、どう伝えるかを考えながら、案内として自然な構成に整えています。

開発だけでなく提案にも関与

実装だけでなく、どう活用できるかを考えながら提案や説明にも関わっています。

VALUE

システムの価値

分かりやすい案内ができる

文章だけでは伝わりにくい内容も、映像と音声を使って直感的に届けられます。

運用しやすい

データを更新することで案内内容を変えられるため、継続的な利用に向いています。

情報発信の幅が広がる

案内・説明・お知らせなど、さまざまな用途に展開しやすい仕組みとして活用できます。
